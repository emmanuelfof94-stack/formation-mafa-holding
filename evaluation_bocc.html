Question 01 / 033 pts
Décrivez les 3 niveaux du processus de validation Connectel. Pour chaque niveau, précisez le rôle de l'acteur concerné et les actions attendues.
✅ Réponse modèle
Niveau 1 — Collecte terrain (Call Center / Agent terrain) : L'agent entre en contact avec l'entrepreneur (appel ou visite). Il présente le programme ID30, collecte les données d'identité (nom, prénom, photo, CNI, téléphone), obtient le consentement et crée le dossier dans Connectel.

Niveau 2 — Vérification administrative (Back Office) : L'agent BO contrôle la complétude (tous documents présents), la lisibilité (photos nettes, documents valides), la cohérence des données et la conformité KYC. Il attribue le statut : Validé, Incomplet ou Rejeté.

Niveau 3 — Validation finale (Superviseur) : Le superviseur effectue un contrôle par échantillonnage. Il valide définitivement les dossiers conformes, les transmet à Orange Money pour activation, gère les litiges et prend les décisions sur les cas complexes.
Question 02 / 033 pts
Un entrepreneur vous dit : « Je ne comprends pas pourquoi vous avez mes coordonnées, c'est de l'espionnage ! » Rédigez votre script de réponse complet en tant qu'agent Call Center.
✅ Réponse modèle
Empathie : « Je comprends tout à fait votre inquiétude, M./Mme [Nom], et je vous remercie de me le signaler. »

Explication : « Vos coordonnées nous ont été transmises dans le cadre du Programme ID30, une initiative officielle du Ministère du Commerce et de l'Industrie, en partenariat avec Orange Money. »

Réassurance : « Vos données sont strictement confidentielles, sécurisées et utilisées uniquement dans ce cadre. »

Reprise ou respect du refus : Si intéressé : « Puis-je vous expliquer comment ce programme peut vous aider ? ». Si refus : « Je comprends. Je note vos préférences. Bonne continuation. »
→ Statut « Refus / Méfiance » dans Connectel.
Question 03 / 033 pts
Quelles sont les 5 règles d'or d'un agent Back Office pour garantir la qualité de traitement des dossiers ? Expliquez brièvement chacune.
✅ Réponse modèle
Règle 1 — Vérifier avant de valider : Ne jamais valider un dossier sans contrôler l'intégralité des pièces. Un dossier validé à tort crée des erreurs en cascade.

Règle 2 — Tout noter dans Connectel : Chaque action (statut, commentaire, demande de correction) doit être tracée dans l'outil pour assurer la traçabilité.

Règle 3 — Respecter les délais : Un dossier reçu doit être traité dans les 48h ouvrables. Les dossiers en attente de reconfirmation doivent être relancés sous 48h.

Règle 4 — Communiquer clairement les corrections : Toute demande de correction doit être précise (quelle pièce, quel problème, délai attendu).

Règle 5 — Escalader les cas complexes : En cas de doute, escalader immédiatement au superviseur sans tenter de décider seul.

Cas 01 — Call Center3 pts
📋 Scénario Vous êtes agent Call Center. Vous appelez Mme Ahou Kouamé. Elle décroche et dit avec une voix irritée : « Encore vous ! C'est la troisième fois cette semaine ! J'ai déjà dit non à vos collègues ! Je vais porter plainte ! »

Contexte Connectel : 2 appels précédents enregistrés : « Sans réponse » et « Intéressée, à rappeler ». Aucune note de refus précédent. Statut actuel : « À contacter ».
1/3 — Quelle est la source du problème ? (1 pt)
2/3 — Rédigez ce que vous dites à Mme Kouamé (1 pt)
3/3 — Actions post-appel dans Connectel (1 pt)
✅ Correction modèle — 3 points
Analyse (1 pt) : Erreur de qualification par le collègue — le 2e appel noté « Intéressée, à rappeler » est faux (elle avait déjà refusé). Source : mauvaise saisie du statut qui génère un 3e appel non souhaité.

Script (1 pt) : « Mme Kouamé, je vous présente toutes mes excuses pour ce dérangement répété. Il y a eu une erreur dans notre système de suivi. Je note immédiatement votre refus définitif et m'assure que vous ne serez plus contactée. Encore désolé. Bonne journée. » — Ton calme, bref, sans se défausser sur le collègue.

Actions Connectel (1 pt) : (1) Statut → « Refus définitif — Ne pas rappeler » ; (2) Note : « Cliente très mécontente — 3 appels en une semaine, qualification précédente incorrecte. Bloquer tout rappel. » ; (3) Signaler au superviseur l'erreur de qualification pour sensibilisation qualité.
Cas 02 — Back Office3 pts
📋 Scénario Vous êtes agent Back Office. Vous recevez le dossier de M. Konan Yves, entrepreneur à Bouaké.

Résultat du contrôle :
✓ Nom / Prénom — Lisible et cohérent
✗ Photo de profil — Floue, visage non identifiable
✗ Numéro de téléphone — Format incorrect (07 58 41 3X)
✓ CNI recto — Lisible, valide jusqu'au 2027
⚠ CNI verso — Absent du dossier
⚠ Coordonnées GPS — Non renseignées
1/3 — Classez les problèmes (bloquant / à corriger / mineur) (1 pt)
2/3 — Quel statut dans Connectel ? Rédigez le message de retour au terrain (1 pt)
3/3 — Délai attendu et procédure si pas de retour (1 pt)
✅ Correction modèle — 3 points
Diagnostic (1 pt) : Bloquants : Photo floue (KYC impossible) + Numéro de téléphone illisible (compte OM impossible à créer). À corriger obligatoirement : CNI verso absent. Mineur : GPS non renseigné (peut être corrigé sans retour terrain en cas d'adresse connue).

Statut + Message (1 pt) : Statut → « Incomplet — À corriger » (pas Rejeté — les données de base sont correctes). Message : « Bonjour, le dossier de M. Konan Yves (Bouaké) présente 3 corrections : (1) Photo floue — refaire une photo nette, fond neutre ; (2) Numéro de téléphone illisible — confirmer le numéro complet (10 chiffres) ; (3) CNI verso absent — photographier le verso. Merci de corriger dans les 48h. Réf : [ID Connectel]. »

Délai & suivi (1 pt) : 48h ouvrables. Si pas de retour : relance message + appel. Si pas de retour à 72h : escalade superviseur. Le dossier reste en statut « Incomplet » jusqu'à correction complète.
